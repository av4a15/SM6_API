Я сделал API для сервиса по ремонту ноутбуков. Проект написан на Django с использованием Django 
REST Framework. В нём можно вести учёт клиентов, брендов ноутбуков, самих устройств, которые
принесли в ремонт, и записей о выполненных ремонтах.

Функционал

Для каждой модели реализованы все основные операции. Можно получить список всех записей или одну 
конкретную по id. Можно создать новую запись — одну или сразу несколько, если передать список.
Можно полностью обновить запись или только отдельные поля. Можно удалить одну запись или несколько
сразу, передав их id через параметр ids. Все GET-запросы кешируются на 15 минут, чтобы не грузить
базу. Есть фильтрация списков по различным полям.

Эндпоинты

Все эндпоинты начинаются с /api/v1/. Для клиентов доступны GET /clients/ со фильтром по имени, GET /
clients/id/, POST /clients/ для создания одного или нескольких, PUT и PATCH для обновления, DELETE 
по id или через ids для массового удаления. Для брендов то же самое — GET /brands/ с фильтром по 
названию, GET /brands/id/, POST, PUT, PATCH, DELETE. Для ноутбуков GET /laptops/ с фильтрами по 
бренду и клиенту, а также все остальные операции. Для ремонтов GET /repairs/ с фильтром по ноутбуку 
и все остальные методы.
